# Lab Overview

## Project Name

IAM Access Governance Lab | Microsoft Entra ID with Okta and SailPoint Concept Mapping

## Repository

iam-access-governance-entra-id-lab

## Purpose

This project is a hands-on IAM access governance lab built in Microsoft Entra ID. The lab demonstrates identity lifecycle management, role-based access control, MFA enforcement, Conditional Access, least-privilege administration, privileged access management, access review concepts, and joiner/mover/leaver workflows.

Okta and SailPoint are included only as conceptual comparisons. This project does not configure live integrations with Okta or SailPoint.

## Lab Environment

Northstar Identity Lab is the Microsoft Entra ID lab tenant used for this project.

The lab uses fictional users, departments, and security groups to model identity and access governance scenarios across HR, Finance, IT, Conditional Access, privileged access, and access review workflows.

## Phase 1 Scope

- Microsoft Entra tenant setup
- Test users
- Security groups
- Role-based access assignments
- MFA and Conditional Access
- Joiner, mover, and leaver workflows
- Least-privilege admin role assignment
- Privileged Identity Management
- Manual access review simulation

## Phase 2 Scope

Microsoft Entra ID Governance may be added later to demonstrate access packages, Lifecycle Workflows, automated access requests, approvals, and more advanced access review scenarios.

## Important Note

This lab is built hands-on in Microsoft Entra ID only. Okta and SailPoint are discussed only for conceptual IAM mapping.
